UC Davis — Spring 2026

This practical reference for data analysis labs applies to all labs requiring computational work.

1. Which tool should I use?

You may use any programming language and any AI coding tool; the deliverable remains the same regardless. MATLAB is the safe default, as all UC Davis students have access via the campus license, which includes MATLAB Copilot.

2. The non-negotiable rule

You are responsible for every line of code you submit.
AI tools can produce incorrect results in subtle ways. In engineering, code you cannot explain is code you cannot trust.

Every submission including AI-generated code must contain:
1. The code itself: Complete and runnable, with a comment at the top naming the tool used (e.g., % Generated with MATLAB Copilot, adapted by [your name]).
2. The output: The actual plot or table produced by running that code on your specific data.
3. A brief code review: 3–5 sentences written by you (not the AI) explaining:
* What the code does step-by-step.
* One thing you verified or would change.
* Whether the output matches physical expectations and why.

3. How to write a prompt that works

Vague prompts are the primary cause of AI failure. A good prompt includes four parts:
* What you have: Describe data names, units, size, and format.
* What you want: Describe the output (plot, table, etc.).
* Any constraints: Language or specific functions to use/avoid.
* A verification request: Ask the AI to explain the code it just generated.

Example: Good Prompt

"I have two MATLAB vectors: t (time in seconds, length 500) and v (normalized voltage, values 0 to 1, from one RC discharge curve). Write MATLAB code that: (1) finds the index where v first drops below 0.368, (2) reads the corresponding time from t and stores it as tau, (3) plots v vs t with a horizontal dashed line at v = 0.368 and a vertical dashed line at t = tau, both labeled. After generating the code, explain what each block does in plain English."

4. When the code does not work

  1. Paste the error back: Send the full error message and line number back to the AI in the same conversation.
  2. Check data format: Verify CSV imports, vector shapes (row vs. column), and normalization.
  3. Use Appendix A: If AI fails three times, adapt the working example code provided in Appendix A of the lab document.

5. Verifying the output

  • Physical Sense: For an RC circuit ($R = 2.2\text{ k}\Omega$, $C = 1\text{ }\mu\text{F}$), $\tau$ should be $\approx 2.2\text{ ms}$.
  • Visual Check: Normalized voltage should span 0 to 1; threshold lines should cross the curve correctly.
  • Consistency: Investigate if per-cycle $\tau$ values vary by more than 20%.

7. Academic integrity

Using AI is encouraged, but submitting output you do not understand is prohibited. AI as a thinking partner is the goal; AI as a shortcut that bypasses thinking is not.
